# playbooks/root-cause-analysis

## Purpose
Templates and workflows for conducting post-incident root cause analyses (RCAs).

## What Belongs Here
- RCA templates and timelines.
- Question prompts for blameless post-mortems.
- Follow-up action tracking formats.

## What Does NOT Belong Here
- General debugging procedures (use `debugging/`).
- Project specs or design docs (use `projects/`).

## Lifecycle
1. Initiate an RCA immediately after an incident is resolved.
2. Use the template to gather facts and build a timeline.
3. Publish findings and track action items to completion.

## Example Entry
- `rca-template.md`: A standard template for documenting incident timelines, root causes, and remediation actions.
