# playbooks/project-start

## Purpose
Checklists and templates for initiating new engineering projects.

## What Belongs Here
- Project kickoff checklists.
- Repository setup and scaffolding steps.
- Initial documentation and onboarding guides.

## What Does NOT Belong Here
- Active project documentation (use `projects/`).
- Generic coding standards (use `rules/coding/`).

## Lifecycle
1. Use the checklist when a new project is approved.
2. Customize for the specific project context.
3. Archive the checklist with the project in `projects/archived/`.

## Example Entry
- `new-microservice-checklist.md`: A checklist covering repo creation, CI setup, observability, and initial docs.
