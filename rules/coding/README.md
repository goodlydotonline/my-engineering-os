# rules/coding

## Purpose
Coding standards, style guides, and language-specific conventions.

## What Belongs Here
- Language style guides (Go, Python, TypeScript, etc.).
- Linting and formatting configurations.
- Naming conventions and directory structure rules.

## What Does NOT Belong Here
- General engineering practices (use `knowledge/engineering/`).
- AI-specific prompts (use `prompts/`).

## Lifecycle
1. Define standards based on language best practices and team consensus.
2. Enforce via linters and code review.
3. Update as languages and tools evolve.

## Example Entry
- `go-style-guide.md`: Conventions for Go code in this organization, including error handling and testing patterns.
