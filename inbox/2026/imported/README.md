# inbox/2026/imported

## Purpose
Files imported from external systems (e.g., exports, downloads, email attachments) awaiting processing.

## What Belongs Here
- CSV/JSON exports from tools.
- Downloaded whitepapers or PDFs.
- Email attachments or chat exports.

## What Does NOT Belong Here
- Files generated by internal workflows (those belong in `projects/` or `automation/`).
- Duplicate imports; deduplicate on arrival.

## Lifecycle
1. Import file with a `YYYY-MM-DD-` prefix.
2. Inspect and extract relevant information.
3. Move extracted data to the appropriate knowledge or project directory.
4. Delete or archive the imported file once processed.

## Example Entry
- `2026-06-19-terraform-state-export.json`: An exported Terraform state file for analysis before migration.
