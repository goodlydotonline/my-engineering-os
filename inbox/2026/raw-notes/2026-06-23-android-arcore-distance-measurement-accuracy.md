# Android ARCore 测距精度与工程实现要点

## 背景

在 Android 端开发 AR 测距功能时，经常会被产品问到“能不能做到 1～2cm 精度”。直接承诺这个指标是有风险的。苹果官方 Measure App 实际上也是通过摄像头扫描环境、用户手动标定起点/终点来完成测量，并且根据机型差异提供 Ruler View 等增强能力。

Android 阵营设备碎片化更严重，精度受多因素影响，因此需要从技术和工程角度理解：精度到底由什么决定，以及如何设计产品、交互与实现方案来逼近目标。

## 涉及的技术点

| 技术点 | 说明 |
|--------|------|
| **ARCore** | Google 的 Android AR SDK，提供运动追踪、环境理解、光照估计、深度估计等能力。 |
| **Depth API** | ARCore 提供的深度感知能力，部分设备支持（通过 ToF、双目、AI 深度估计等）。 |
| **ToF（Time of Flight）** | 飞行时间深度传感器，可直接获取场景深度，精度通常优于纯视觉估计。 |
| **单目视觉 SLAM / 运动追踪** | 普通手机仅靠单个 RGB 摄像头，通过特征点跟踪和运动恢复结构来估计相机位姿和稀疏 3D 结构。 |
| **平面识别（Plane Detection）** | ARCore 识别水平/垂直平面，测距通常依赖平面锚点（Anchor）放置。 |
| **锚点漂移（Anchor Drift）** | 随着时间或相机移动，Anchor 在世界坐标系中的位置发生偏移，是精度下降的主要来源之一。 |
| **Ruler View（iOS 概念）** | 苹果在支持 LiDAR 的机型上提供更直观的标尺视图；Android 端可类比为基于 Depth API 的可视化测量辅助。 |

## 影响精度的关键因素

### 硬件层面
- **手机型号**：是否支持 ARCore，是否支持 Depth API。
- **是否有 ToF / 深度传感器**：直接决定能否获得较准确的深度信息。
- **摄像头质量**：传感器尺寸、镜头畸变、帧率、对焦稳定性。

### 环境层面
- **光照**：过暗、过曝、强光闪烁都会影响特征点提取。
- **墙面/地面纹理**：纯色、白墙、反光面、玻璃等低纹理或高光表面难以稳定识别平面和特征。
- **物体反光/透明**：会导致深度估计错误和特征丢失。
- **距离远近**：越远，单目估计的相对误差越大。

### 用户行为层面
- **用户移动幅度**：移动过快或抖动大会导致追踪丢失、漂移增大。
- **环境扫描充分性**：ARCore 需要足够视角变化来建立稳定地图。

### 算法层面
- **平面识别质量**：平面拟合误差会直接影响基于平面的测量。
- **锚点漂移**：长时间放置的 Anchor 可能因累积误差而漂移。

## 应用场景

- **室内短距离测量**：如测量家具尺寸、墙面挂画位置、门框高度等。
- **现场快速估算**：装修、维修、物流等场景下的粗略尺寸参考。
- **AR 标注与对齐**：在真实空间中放置虚拟标尺或对齐辅助线。
- **垂直行业工具**：建筑、工业设计、电商 AR 预览等。

## 工程实现建议

### 1. 明确精度预期
- 普通安卓手机（单目 + ARCore 运动追踪）：短距离、良好条件下可达到厘米级演示效果，但**不应承诺稳定 < 1～2cm**。
- 支持 Depth API / ToF 的设备：可接近 1～2cm，但仍受环境和使用方法影响。

### 2. 设备能力分级
在 App 中根据设备能力提供不同测量模式：

| 设备能力 | 建议模式 |
|----------|----------|
| 不支持 ARCore | 提示不支持或降级为手动输入 |
| 支持 ARCore，无 Depth API | 基础单目测距，适合演示和粗略估算 |
| 支持 ARCore + Depth API | 启用深度增强测距，提示精度更高 |
| 支持 ToF / 专用深度传感器 | 最佳测距体验 |

### 3. 交互设计
- **引导用户扫描环境**：提示用户缓慢移动手机，让 ARCore 充分建立环境理解。
- **手动标定起点/终点**：参考 Measure App，让用户在屏幕上点击放置锚点。
- **多次测量取平均**：对同一距离进行多次采样，降低随机误差。
- **实时反馈置信度**：根据追踪状态、平面质量、深度质量显示置信度或颜色提示。

### 4. 技术实现要点
- **使用 `Session` 和 `Frame` 获取相机位姿与点云/深度图**。
- **使用 `Plane` 和 `Anchor` 放置测距端点**。
- **优先在 Depth API 可用时使用 `DepthImage` 获取局部深度**，用于修正端点深度。
- **对端点坐标进行滤波**（如卡尔曼滤波、指数移动平均）以减少抖动。
- **监听 `TrackingState` 和 `TrackingFailureReason`**，在追踪丢失时暂停测量并提示用户。

### 5. 标定与补偿
- **现场标定**：在已知长度的参考物上做一次校准，用于补偿当前设备的系统误差。
- **镜头畸变补偿**：必要时使用相机标定参数修正图像坐标。
- **温度/IMU 偏差补偿**：长时间使用可能引入 IMU 漂移，可通过视觉重定位进行修正。

### 6. 测试与验证
- 覆盖不同机型（支持/不支持 Depth API）。
- 覆盖不同环境（光照、纹理、反光、远距离）。
- 对比真实卷尺或激光测距仪，统计误差分布。
- 记录失败案例，反哺规则库和提示文案优化。

## 结论

1～2cm 精度**不能直接承诺**。工程上应将测距功能按设备能力分级，普通安卓手机作为“演示/估算”级别，支持 Depth API / ToF 的设备才能追求更高精度。通过合理的交互引导、多次采样、深度增强和现场标定，可以在可控场景下逼近目标。
