# meta

## Purpose
Meta-documentation about the engineering OS itself: its structure, conventions, and evolution.

## What Belongs Here
- Changelog for the directory structure and conventions.
- Proposals for structural changes.
- Usage statistics and health checks.

## What Does NOT Belong Here
- Project or domain knowledge (use `knowledge/`).
- Operational runbooks (use `playbooks/`).

## Lifecycle
1. Document changes when the structure or conventions evolve.
2. Review periodically to ensure the OS remains useful.
3. Propose and discuss major changes before implementation.

## Example Entry
- `changelog.md`: A log of changes to the engineering OS structure and conventions.
