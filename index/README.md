# index

## Purpose
Indexes, maps, and navigation aids for the engineering OS.

## What Belongs Here
- Directory indexes and glossaries.
- Cross-reference maps between domains.
- Search and discovery aids.

## What Does NOT Belong Here
- Content documents (use `knowledge/` or `projects/`).
- Unstructured notes (use `inbox/`).

## Lifecycle
1. Create indexes as the directory structure grows.
2. Update when new domains or major topics are added.
3. Keep concise and focused on navigation.

## Example Entry
- `knowledge-map.md`: A map linking knowledge domains to key documents and projects.
