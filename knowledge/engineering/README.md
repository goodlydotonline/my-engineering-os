# knowledge/engineering

## Purpose
General software engineering practices, tools, and workflows.

## What Belongs Here
- Development environment setup and tooling notes.
- Testing strategies and CI/CD best practices.
- Code review guidelines and team conventions.

## What Does NOT Belong Here
- Domain-specific knowledge (use `backend/`, `ai/`, etc.).
- Project-specific operational docs (use `projects/`).

## Lifecycle
1. Capture team conventions and tooling lessons.
2. Generalize into reusable guides.
3. Update as tools and practices evolve.

## Example Entry
- `trunk-based-development.md`: A guide on implementing trunk-based development with feature flags.
