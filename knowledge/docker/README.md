# knowledge/docker

## Purpose
Reference material on containerization with Docker.

## What Belongs Here
- Dockerfile best practices and patterns.
- Image optimization and layer caching strategies.
- Compose configurations and local development setups.

## What Does NOT Belong Here
- Kubernetes-specific orchestration docs (use `kubernetes/`).
- CI/CD pipeline definitions (use `playbooks/deployment/`).

## Lifecycle
1. Capture patterns from project work.
2. Generalize into reusable guides.
3. Update as Docker and the ecosystem evolve.

## Example Entry
- `multi-stage-builds.md`: A guide on writing efficient multi-stage Dockerfiles for Go services.
