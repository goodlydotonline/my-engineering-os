# knowledge/kubernetes

## Purpose
Reference material on Kubernetes, container orchestration, and cloud-native operations.

## What Belongs Here
- Manifest patterns, Helm charts, and Kustomize configurations.
- Cluster management and security hardening notes.
- Debugging and observability within K8s.

## What Does NOT Belong Here
- Docker-specific build docs (use `docker/`).
- General cloud provider docs (use `architecture/` or `engineering/`).

## Lifecycle
1. Capture operational lessons and patterns.
2. Generalize into reusable guides.
3. Update as Kubernetes versions and best practices change.

## Example Entry
- `pod-disruption-budgets.md`: A guide on configuring PDBs for zero-downtime deployments.
