# knowledge/ai

## Purpose
Reference material on artificial intelligence, machine learning, and LLM usage.

## What Belongs Here
- Model selection guides and benchmarking notes.
- Prompt engineering patterns and evaluation methods.
- Integration patterns for AI services.

## What Does NOT Belong Here
- Raw chat logs or unprocessed model outputs.
- Generic software engineering patterns (use `engineering/`).

## Lifecycle
1. Document experiments and findings.
2. Distill into reusable guides.
3. Keep updated as models and tools evolve.

## Example Entry
- `prompt-caching-strategies.md`: Notes on optimizing prompt caching with the Anthropic API.
