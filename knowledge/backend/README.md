# knowledge/backend

## Purpose
Reference material for backend engineering, APIs, databases, and server-side patterns.

## What Belongs Here
- API design guidelines and patterns.
- Database schema design and optimization notes.
- Service architecture and middleware patterns.

## What Does NOT Belong Here
- Frontend or mobile-specific knowledge.
- Infrastructure provisioning docs (use `docker/` or `kubernetes/`).

## Lifecycle
1. Capture lessons from projects and incidents.
2. Generalize into reusable patterns.
3. Update as technologies and best practices evolve.

## Example Entry
- `idempotency-keys.md`: A guide on implementing idempotency keys for payment APIs.
