# projects/archived

## Purpose
Storage for completed, cancelled, or deprecated projects.

## What Belongs Here
- Final versions of project documentation.
- Post-mortems and retrospectives.
- Historical decision records for reference.

## What Does NOT Belong Here
- Active project work.
- General knowledge that is still current (move to `knowledge/` instead).

## Lifecycle
1. Move a project directory here when it is no longer active.
2. Keep the directory read-only; do not add new work.
3. Retain for historical reference; delete only after a defined retention period.

## Example Entry
- `2025-legacy-migration/`: Final docs and post-mortem for a completed migration.
