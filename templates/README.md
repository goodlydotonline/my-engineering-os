# templates

## Purpose
Reusable templates for documents, issues, pull requests, and other recurring artifacts.

## What Belongs Here
- Markdown templates for RFCs, ADRs, and post-mortems.
- Issue and PR templates.
- Meeting notes and retrospective templates.

## What Does NOT Belong Here
- Final, filled-in documents (use `projects/` or `knowledge/`).
- One-off drafts.

## Lifecycle
1. Create a template when a document type is repeated.
2. Use the template for new instances.
3. Refine the template based on feedback.

## Example Entry
- `adr-template.md`: A template for Architectural Decision Records.
